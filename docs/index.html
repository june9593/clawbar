Get your OpenClaw in the menu bar in 30 seconds.

One download, one drag (or one Next-Next-Install). Point it at your gateway URL and you're in — on macOS or Windows.

macOS (.dmg) v0.4.0 · arm64 Windows (.exe) v0.4.0 · x64 All releases →

Install — macOS (Apple Silicon)

1Download ClawBar-0.4.0-mac-arm64.dmg.
2Open it and drag ClawBar into /Applications.
3First launch — right-click → Open, then "Open" in the Gatekeeper dialog. The app isn't code-signed.
4Or skip step 3: 
xattr -dr com.apple.quarantine /Applications/ClawBar.app

Install — Windows (x64)

1Download ClawBar-0.4.0-win-x64.exe.
2Double-click to install. SmartScreen warns (installer isn't signed) — click More info → Run anyway.
3Launch from Start Menu or desktop shortcut. The lobster appears in the system tray — on Windows 11, drag it out of the overflow flyout to pin it.

Build from source

# clone, install, run (macOS or Windows)
git clone https://github.com/june9593/clawbar.git
cd clawbar
npm install
npm run dev:electron

# package your own build
npm run pack:mac:dmg:arm64    # macOS
npm run pack:win              # Windows (NSIS + portable)
Frequently asked

Sensible questions, short answers.

Why isn't the build signed?

Code signing costs money on both platforms — Apple Developer is $99/yr plus notarisation, and Windows EV / OV certs are $200–$500/yr. ClawBar is a free side-project, so the DMG and the NSIS installer are both unsigned. On macOS you bypass Gatekeeper once with right-click → Open (or xattr -dr com.apple.quarantine); on Windows you click More info → Run anyway in the SmartScreen dialog. Both are one-time.

Does it work on Intel Macs?

Not in releases — we ship macOS arm64 only. The build pipeline can produce x64 (run npm run pack:mac:dmg:x64 from source), it's just not part of the GitHub Actions release. PRs welcome.

Does the Windows build work on Windows 10 / ARM?

Releases ship Windows x64, which covers Windows 10 22H2 and Windows 11 on Intel/AMD. ARM64 Windows and 32-bit Windows aren't built by the release pipeline — you can produce them from source by editing electron-builder.yml's win.target.arch. On Windows 11, the tray icon often starts hidden in the overflow flyout — drag it onto the main taskbar to pin it.

What's the OpenClaw connection like?

ClawBar talks to your gateway over a single WebSocket from the Electron main process. Ed25519 device identity, generated and stored locally at ~/.clawbar/device-identity.json — the private key never leaves the main process. Tokens never reach the renderer either.

Does it phone home?

No. No analytics, no error reporting, no auto-update check. The only network calls are the ones you make: the WebSocket to your OpenClaw gateway, and whatever each <webview> channel does on its own.

Where are settings stored?

~/.clawbar/settings.json for app config (gateway URL, theme, channel list, pet visibility). ~/.clawbar/device-identity.json for the Ed25519 keypair. Each channel's cookies and localStorage live in Electron partitions under your Electron user-data directory.

Can I add a channel that isn't built in?

Yes — anywhere your OpenClaw bot is reachable as a web URL works. Click + at the bottom of the bar, paste the URL. The favicon auto-loads as the icon, hostname becomes the name. Right-click to rename, reorder, or delete. Custom Mattermost, internal chat tools, anything web-based.

Do I need OpenClaw to use ClawBar?

The first channel (the lobster) is the OpenClaw native chat — that part assumes you have an OpenClaw gateway running and reachable. The other channels (Telegram / Discord / Feishu / Lark / custom URLs) are just web wrappers, so they work standalone too. But the project is built around OpenClaw — that's the agent on the other end of all these channels.

Is it open source?

Yes — MIT licensed. Source at github.com/june9593/clawbar. Issues and PRs welcome.
